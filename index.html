Introducción

Este proyecto tiene como objetivo crear un generador de exámenes de ecuaciones diferenciales que busca facilitar a los maestros la creación de exámenes y ahorrarles tiempo. Esto se logra gracias a un banco de preguntas en CSV, el cual tiene diversas secciones para adaptarse a las necesidades del usuario.

Requisitos

Funcionales

  • El usuario podrá crear una cuenta con nombre de usuario y contraseña.
  • El usuario podrá generar un examen aleatorio con su banco de preguntas personalizado.
  • El usuario podrá crear un examen personalizado por tema.
  • El usuario podrá crear un examen personalizado por dificultad.

No Funcionales

  • El sistema debe ser fácil de usar y accesible.
  • El sistema debe ser rápido y eficiente.

Conclusión

Este proyecto nos a habierto el panorama sobre el como podemos implementar lo aprendido en el curso en algo util y que las personas puedan usar en su dia a dia. El proyecto nos ayudo a poder entender mejor los temas vistos en clase como el uso de punteros para manipular datos de manera eficiente, estructuras (struct) para organizar la información, memoria dinámica para gestionar recursos en tiempo de ejecución, recursión para resolver problemas de forma modular, y manejo de archivos para almacenar y recuperar datos de manera persistente.

Codigo

Descripción del Código

Este código implementa un generador de exámenes aleatorios en C, utilizando una tabla que almacena preguntas. El programa permite cargar un archivo CSV con preguntas, filtrar por categorías, elegir la cantidad de preguntas por dificultad y generar un examen en un archivo de texto. El sistema emplea estructuras de datos para representar las tablas y filas de preguntas, y permite la manipulación dinámica de la información.

Estructuras de Datos

Tabla

La estructura TABLE representa una tabla que almacena varias filas, cada una de ellas con información sobre las preguntas del examen.

  • numCat: Número de categorías de preguntas disponibles.
  • CAT: Un arreglo de cadenas que contiene los nombres de las categorías.
  • firstROW: Puntero a la primera fila de la tabla.

Fila

La estructura ROW representa una fila de la tabla, que contiene las celdas de datos para una pregunta en particular.

  • firstCell: Puntero a la primera celda de la fila.
  • next: Puntero a la siguiente fila de la tabla.

Celda

La estructura CELL representa una celda individual dentro de una fila, que contiene un dato específico de la pregunta.

  • data: El valor o contenido de la celda (por ejemplo, texto de la pregunta o respuesta).
  • nextRow: Puntero a la siguiente celda en la misma columna.
  • nextColumn: Puntero a la siguiente celda en la misma fila, en otra columna.

Flujo del Programa

El flujo del programa es el siguiente:

  1. Se muestra un mensaje de bienvenida e inicio de sesión.
  2. El usuario puede elegir entre las diferentes opciones del menú para cargar un archivo CSV, elegir temas, seleccionar la cantidad de problemas y generar el examen.
  3. Al generar el examen, se crea un archivo de texto con las preguntas seleccionadas.
  4. Finalmente, el programa guarda las cuentas de usuario y limpia la memoria dinámica antes de finalizar.

1.- Cuenta.h

Descripción del Código

El código implementa un sistema básico de gestión de cuentas de usuario en C, utilizando estructuras de datos y funciones para permitir la creación, inicio de sesión, carga y guardado de cuentas. Se emplea un archivo de texto para almacenar las cuentas de usuario de manera persistente.

Estructuras de Datos

Cuenta

La estructura Cuenta representa una cuenta de usuario, que incluye el nombre de usuario y la contraseña asociados.

  • usuario: Nombre del usuario, almacenado como una cadena de caracteres (char).
  • contra: Contraseña del usuario, almacenada como una cadena de caracteres (char).

Funciones

Función Descripción
menu Muestra el menú principal de opciones al usuario.
menu_sesion Controla la interacción con el usuario, permitiendo crear cuentas o iniciar sesión.
crear_cuenta Permite al usuario crear una nueva cuenta de usuario.
login Verifica si el nombre de usuario y la contraseña coinciden con una cuenta registrada.
guardar_cuentas Guarda las cuentas de usuario en un archivo de texto.
cargar_cuentas Carga las cuentas de usuario desde un archivo de texto.

Ejemplo de Uso


                            int main(int argc, char const * argv[])
                            {
                                // Iniciar sesión
                                inicio_sesion();
                                return 0;
                            }
                    
                            void inicio_sesion()
                            {
                                Cuenta *cuentas = NULL;
                                int total_cuentas = 0;
                    
                                // Cargar cuentas desde el archivo
                                cargar_cuentas(&cuentas, &total_cuentas);
                    
                                // Iniciar el menú de sesiones

                                // Liberar memoria dinámica
                                free(cuentas);    
                            }

2.- Tablas.h

Descripción del Código

La librería está diseñada para manejar tablas dinámicas basadas en una estructura flexible y eficiente que encapsula filas, celdas y categorías. Las estructuras principales son TABLE, ROW y CELL, las cuales trabajan en conjunto para representar tablas multidimensionales donde cada celda puede contener distintos tipos de datos.

Estructuras de Datos

CELL

Representa una celda dentro de la tabla. Cada celda contiene un dato, un puntero a la siguiente celda en la misma fila (horizontalmente) y un puntero a la siguiente fila (verticalmente).

  • data: Dato almacenado en la celda, que se obtiene desde un archivo CSV. Los datos se guardan como cadenas de caracteres (char *).
  • nextColumn: Puntero a la siguiente celda en la misma fila.
  • nextRow: Puntero a la siguiente celda en la misma columna (fila siguiente).
  • cat: Etiqueta o categoría que describe el tipo de dato que contiene la celda.

ROW

Representa una fila en la tabla, que contiene un conjunto de celdas.

  • firstCell: Puntero a la primera celda de la fila.
  • next: Puntero a la siguiente fila en la tabla.

TABLE

Representa la tabla completa, que contiene varias filas.

  • firstROW: Puntero a la primera fila de la tabla.
  • auxPtr: Arreglo de punteros a las celdas en cada columna para enlazarlas verticalmente.
  • numCat: Número de categorías (columnas) en la tabla.
  • CAT: Arreglo de cadenas de caracteres que contienen los nombres de las categorías (columnas) en la tabla.

Resumen de la estructura de datos

La estructura TABLE encapsula la tabla y permite acceder a las filas y categorías.

Las filas (ROW) son accesibles secuencialmente a través del puntero next, y cada fila contiene un mapa de celdas.

Las celdas (CELL) están conectadas horizontalmente (nextColumn) y verticalmente (nextRow), formando una estructura que permite navegar toda la tabla eficientemente.

Cada celda almacena datos en formato char*, con una etiqueta de categoría que define cómo debe interpretarse el dato.

Ejemplo de Navegación de la Tabla

  • Desde TABLE->firstROW se accede a la primera fila.
  • Desde ROW->firstCell se navega por las celdas de la fila usando nextColumn.
  • Para moverse a la siguiente fila, se utiliza ROW->next.
  • Para navegar por una columna completa, se utiliza CELL->nextRow.

Esta arquitectura flexible permite:

  • Representar tablas de cualquier tamaño.
  • Almacenar datos heterogéneos con diferentes categorías.
  • Navegar la tabla horizontal y verticalmente de manera eficiente.
  • Facilitar la manipulación de datos provenientes de archivos como CSV.

Funciones

Función Descripción
createTable Crea una nueva tabla con un número específico de categorías (columnas).
createRow Crea una nueva fila vacía.
addRow Añade una nueva fila a la tabla.
createCell Crea una nueva celda con un dato específico y una categoría.
addCell Añade una celda en una fila de una tabla existente.

                            // Imprimir la columna "Problema" de la tabla filtrada en un archivo de texto
                            printColumnToTxT(t_1, "Problema");
                        }

Funciones importantes

upload_CSV_Table

Descripción: Esta función carga el contenido de un archivo CSV en una estructura de tabla (TABLE).

Parámetros:

  • TABLE *t: Puntero a la tabla donde se almacenará la información del CSV.
  • const char *fileName: Nombre o ruta del archivo CSV a cargar.

Funcionamiento:

La función lee el archivo línea por línea, tokeniza los datos separados por comas y los almacena en la estructura de la tabla, eliminando caracteres de salto de línea y asignando etiquetas según corresponda.

void upload_CSV_Table(TABLE *t, const char *fileName);

printTable

Descripción: Imprime el contenido de una tabla en formato tabular en la consola.

Parámetros:

  • TABLE *t: Puntero a la tabla que se imprimirá.

Funcionamiento:

Itera sobre las filas y columnas de la tabla y las imprime formateadas con separadores visuales.

void printTable(TABLE *t);

rowCopy

Descripción: Copia el contenido de una fila específica en una nueva fila dentro de la tabla.

Parámetros:

  • TABLE *t: Puntero a la tabla donde se añadirá la nueva fila.
  • ROW *cRow: Puntero a la fila que será copiada.
ROW *rowCopy(TABLE *t, ROW *cRow);

filterByIntCat

Descripción: Filtra las filas de una tabla según una condición aplicada a una categoría numérica.

Parámetros:

  • TABLE *t: Puntero a la tabla que será filtrada.
  • char *cat: Nombre de la categoría numérica a filtrar.
  • int filterType: Tipo de comparación (1 = mayor que, 2 = menor que, etc.).
  • int com: Valor con el que se comparará cada dato.
TABLE *filterByIntCat(TABLE *t, char *cat, int filterType, int com);

filterByStringCat

Descripción: Filtra las filas de una tabla según una condición aplicada a una categoría de texto.

Parámetros:

  • TABLE *t: Puntero a la tabla que será filtrada.
  • char *categ: Nombre de la categoría de texto a filtrar.
  • char **cat: Lista de valores aceptados para la categoría.
TABLE *filterByStringCat(TABLE *t, char *categ, char **cat);

compare_filterauxiliar

Descripción: Compara dos valores enteros según un tipo de comparación.

Parámetros:

  • int a: Primer valor.
  • int b: Segundo valor.
  • int filterType: Tipo de comparación (1 = mayor que, 2 = menor que, etc.).
int compare_filterauxiliar(int a, int b, int filterType);

Inicio

Para comenzar a usar el generador, primero debes de iniciar sesion, si no tienes una cuenta creada, te vas al apartado que dice crear cuenta y la creas. Una vez creada la cuenta inicias sesion con el usuario y contraseña antes creadas.

Uso

Una vez hallas iniciado sesion, se desplegara un menu el cual podras interactuar seleccionando el numero que quieras de este

Explicación de las opciones

[1] CARGAR CSV

Esta opción permite cargar un archivo CSV que contiene el banco de preguntas. El archivo debe seguir el formato:

  • ID: Identificador único de la pregunta.
  • Problema: Texto de la pregunta.
  • Dificultad: Nivel de dificultad (1 a 5).
  • Peso: Peso asignado a la pregunta.
  • Tema: Categoría o tema de la pregunta.

[2] ELEGIR LOS TEMAS

Permite al usuario seleccionar los temas específicos que desea incluir en el examen. Los temas deben coincidir con los nombres de las categorías en el archivo CSV cargado. Se ingresan de manera manual, uno por uno.

[3] ELEGIR LA CANTIDAD DE PROBLEMAS

En esta opción, el usuario define cuántos problemas desea incluir en el examen para cada nivel de dificultad. El programa muestra la cantidad disponible de preguntas por dificultad y valida que el número ingresado no exceda el total disponible.

[4] GENERAR EXAMEN

Genera un examen aleatorio basado en los temas y la cantidad de preguntas seleccionadas. El examen se guarda en un archivo de texto llamado Examen.txt, con las preguntas listadas por orden.

[5] LIMPIAR TABLA

Reinicia la tabla de preguntas, eliminando cualquier información previamente cargada. Esto permite comenzar de nuevo con un nuevo banco de preguntas.

[0] SALIR DEL PROGRAMA

Finaliza la ejecución del programa y cierra la aplicación.
